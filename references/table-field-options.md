# 飞书多维表格字段结构（Boss直聘候选人招聘管理）

> `base_token` 和 `table_id` 存储在 `config.json` 中，通过 `boss init` 自动生成或手动填写。

---

## 字段列表

| 字段名 | 飞书字段类型 | type值 | 说明 |
|--------|-------------|--------|------|
| 姓名 | 文本 | 1 | 默认字段，自动存在 |
| 应聘职位 | 单选 | 3 | 选项由 `config.positions` 决定，初始化时写入 |
| 性别 | 单选 | 3 | 男 / 女 |
| 年龄 | 数字 | 2 | 整数 |
| 学历 | 单选 | 3 | 博士 / 硕士 / 本科 / 大专 |
| 毕业院校 | 文本 | 1 | |
| 专业 | 文本 | 1 | |
| 工作年限 | 数字 | 2 | 整数，应届生填 0 |
| 期望城市 | 文本 | 1 | |
| 期望薪资 | 文本 | 1 | |
| 活跃状态 | 单选 | 3 | 见活跃状态映射表 |
| 近期工作经历 | 文本 | 1 | 格式：公司·职位；公司·职位 |
| 初筛评级 | 单选 | 3 | 见评级选项表 |
| 初筛理由 | 文本 | 1 | |
| 跟进状态 | 单选 | 3 | 默认写入"待联系" |
| 排除原因 | 文本 | 1 | |
| 录入日期 | 日期 | 5 | 格式 `yyyy-MM-dd` |
| Boss直聘消息摘要 | 文本 | 1 | |

---

## 活跃状态映射

boss-cli 返回值与多维表格选项值不完全一致，写入时需映射：

| boss-cli 返回值 | 写入多维表格的值 |
|----------------|----------------|
| 刚刚活跃 | 刚刚活跃 |
| 今日活跃 | 今日活跃 |
| 昨天 | 昨天 |
| 3日内活跃 | **3天内** |
| 本周内 | 本周内 |

## 批量写入数据格式

lark-cli 使用简化格式（非标准飞书 API 格式）：

```json
{
  "fields": ["姓名","应聘职位","性别","年龄","学历","毕业院校","专业","工作年限",
             "期望城市","期望薪资","活跃状态","近期工作经历","初筛评级","初筛理由",
             "跟进状态","排除原因","录入日期","Boss直聘消息摘要"],
  "rows": [
    ["张三","APP软件产品经理","男",30,"本科","某大学","软件工程",5,
     "深圳","20-30K","刚刚活跃","A公司·产品经理","⭐⭐⭐⭐ 推荐","5年经验，本科",
     "待联系","","2026-04-28","您好"]
  ]
}
```

## 常见写入错误

| 错误 | 原因 | 解决 |
|------|------|------|
| `not_found` | 应聘职位写入了不在选项中的值 | 对照 `config.positions`，或在飞书手动添加选项 |
| `Provide a number value` | 年龄/工作年限传了字符串 | 改为纯数字 |
| `Provide a value of type array` | 使用了标准飞书 API 格式 `{"records": [...]}` | 改用 lark-cli 简化格式 |
